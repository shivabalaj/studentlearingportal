# Full-Stack Course App Deployment to Render.com&#10;&#10;This project has a Node.js/Express backend API and React frontend. Deploy separately on Render.com.&#10;&#10;## Prerequisites&#10;- MongoDB Atlas account (free cluster)&#10;- GitHub repo with this code&#10;&#10;## Backend Deployment (Web Service)&#10;1. Go to render.com, New > Web Service, connect GitHub repo&#10;2. Root Directory: `backend`&#10;3. Runtime: Node&#10;4. Build Command: `npm install`&#10;5. Start Command: `npm start`&#10;6. Environment Variables:&#10;   - `MONGO_URI`: Your Atlas connection string (replace <user>:<pass> etc.)&#10;   - `JWT_SECRET`: Secure random string&#10;7. Deploy&#10;&#10;## Frontend Deployment (Static Site)&#10;1. New > Static Site, connect same repo&#10;2. Root Directory: `frontend`&#10;3. Build Command: `npm install &amp;&amp; npm run build`&#10;4. Publish directory: `build`&#10;5. Deploy&#10;&#10;## Database Setup&#10;1. Create MongoDB Atlas cluster (free M0)&#10;2. Create user/database&#10;3. Get connection string: mongodb+srv://...&#10;4. Network Access: Add 0.0.0.0/0 (allow all)&#10;5. Locally: copy .env.example to .env, set MONGO_URI/JWT_SECRET, `cd backend &amp;&amp; npm i &amp;&amp; node seed-admin.js` to seed data&#10;&#10;## Usage&#10;- Frontend URL from Render (e.g. app.onrender.com)&#10;- Backend API base: https://your-backend.onrender.com/api&#10;- Update frontend API calls if hardcoded localhost (check code, currently proxies/relative)&#10;&#10;## Notes&#10;- Render free backend sleeps after 15min inactivity&#10;- Seed data once after Atlas setup&#10;- For production: Use paid Render, secure Atlas IP whitelist&#10;
